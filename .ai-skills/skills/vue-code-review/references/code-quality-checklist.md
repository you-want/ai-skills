# 代码质量检查清单

## 错误处理

### 需要标记的反模式

- **吞掉异常**：空的 catch 块，或 catch 里只有日志
  ```javascript
  try { ... } catch (e) { }           // 静默失败
  try { ... } catch (e) { console.log(e) }  // 记录后忽略
  ```
- **过于宽泛的 catch**：捕获 `Exception`/`Error` 基类而不是具体类型
- **错误信息泄露**：向用户暴露堆栈或内部细节
- **缺失错误处理**：IO、网络、解析等易错操作没有 try-catch
- **异步错误处理缺失**：未处理的 Promise 拒绝、缺少 `.catch()`、没有错误边界

### 最佳实践检查

- [ ] 错误在合适的边界被捕获
- [ ] 错误提示对用户友好（不暴露内部细节）
- [ ] 错误日志包含足够的调试上下文
- [ ] 异步错误被正确传播或处理
- [ ] 可恢复错误有降级方案
- [ ] 关键错误触发告警/监控

### 自问

- "这个操作失败时会发生什么？"
- "调用方会知道出错了吗？"
- "日志里有足够的上下文来调试吗？"

## 性能与缓存

### CPU 密集型操作

- 热路径中的昂贵操作：正则编译、JSON 解析、循环中的加密运算
- 阻塞主线程：同步 IO、没有 worker/async 的重计算
- 不必要的重复计算：同一计算多次执行
- 缺少记忆化：纯函数被反复调用但结果未缓存

### 数据库与 IO

- **N+1 查询**：循环中逐条查询而不是批量查询
  ```javascript
  // 错误：N+1
  for (const id of ids) {
    const user = await db.query(`SELECT * FROM users WHERE id = ?`, id)
  }
  // 正确：批量
  const users = await db.query(`SELECT * FROM users WHERE id IN (?)`, ids)
  ```
- **缺少索引**：在没建索引的列上查询
- **过度获取**：`SELECT *` 但只需要少量字段
- **没有分页**：把整个数据集加载到内存

### 缓存问题

- 缺少对昂贵操作的缓存（重复 API 调用、DB 查询、计算）
- 缓存没有 TTL：数据永久不过期
- 缓存没有失效策略：数据已更新但缓存未清理
- 缓存键冲突：键的唯一性不足
- 全局缓存用户特定数据：安全/隐私风险

### 内存问题

- 无界集合：数组/Map 无限增长
- 大对象持有引用：阻止垃圾回收
- 循环中的字符串拼接：应使用 join
- 大文件完全加载：应使用流式处理

### 自问

- "这个操作的时间复杂度是多少？"
- "数据量放大 10 倍/100 倍时会怎样？"
- "这个结果值得缓存吗？应该怎么缓存？"
- "能不能批量处理而不是逐个处理？"

## 边界条件

### Null / Undefined 处理

- 缺少空值检查：在可能为 null 的对象上访问属性
- 真值/假值混淆：`if (value)` 在 `0` 或 `""` 是合法值时出错
- 可选链过度使用：`a?.b?.c?.d` 掩盖了结构性问题
- null 与 undefined 混用：没有明确约定

### 空集合

- 空数组未处理：代码假设数组一定有元素
- 空对象边界：`for...in` 或 `Object.keys` 在空对象上的行为
- 首尾元素访问：`arr[0]` 或 `arr[arr.length-1]` 没有长度检查

### 数值边界

- 除以零：除法前缺少检查
- 整数溢出：超出安全整数范围的大数
- 浮点数比较：使用 `===` 而不是误差范围比较
- 负值：本不应为负的索引或计数
- 越界错误：循环边界、数组切片、分页计算

### 字符串边界

- 空字符串未作为边界处理
- 仅空白字符的字符串：通过真值检查但实际为空
- 超长字符串：没有长度限制导致内存/显示问题
- Unicode 边界：表情符号、RTL 文本、组合字符

### 常见危险模式

```javascript
// 危险：没有空值检查
const name = user.profile.name

// 危险：没有检查的数组访问
const first = items[0]

// 危险：除以零
const avg = total / count

// 危险：真值检查排除了合法值
if (value) { ... }  // 对 0、""、false 会误判
```
