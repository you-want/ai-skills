# 移除与迭代计划模板

## 优先级

- [ ] **P0**：立即移除（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：当前迭代移除
- [ ] **P2**：待办列表 / 下次迭代

---

## 现在可安全移除

### 项目：[名称/描述]

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.ts:line` |
| **理由** | 为什么应该移除 |
| **证据** | 未使用（无引用）、已关闭的特性开关、废弃的 API |
| **影响** | 无 / 低 — 没有活跃的调用方 |
| **移除步骤** | 1. 移除代码 2. 移除测试 3. 移除配置 |
| **验证** | 运行测试、确认无运行时错误、监控日志 |

---

## 延后移除（需要计划）

### 项目：[名称/描述]

| 字段 | 详情 |
|------|------|
| **位置** | `path/to/file.ts:line` |
| **延后原因** | 仍有活跃调用方、需要迁移、需利益相关方确认 |
| **前置条件** | 特性开关关闭 2 周、遥测显示 0 使用 |
| **破坏性变更** | 列出任何 API/契约变更 |
| **迁移计划** | 调用方的迁移步骤 |
| **时间线** | 目标日期或迭代 |
| **负责人** | 负责的个人/团队 |
| **验证** | 确认安全移除的指标（错误率、使用次数） |
| **回滚计划** | 如果发现问题，如何恢复 |

---

## 移除前检查清单

- [ ] 搜索代码库中的所有引用（`rg`、`grep`）
- [ ] 检查动态/反射式使用
- [ ] 确认没有外部调用方（API、SDK、文档）
- [ ] 审查特性开关遥测数据（如适用）
- [ ] 更新/移除相关测试
- [ ] 更新文档
- [ ] 通知团队（如果是共享代码）
